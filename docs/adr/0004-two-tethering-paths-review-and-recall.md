# The knowledge base has two tethering paths: Review and Recall

A loose Memory can become tethered (trusted and retrievable) by **either** of two paths:

- **Review** — a human inspects the memory and asserts it is true/worth trusting. The path for ordinary captured facts (ADR 0001).
- **Recall** — the human proves they *retained* the material by answering spaced recall prompts correctly across multiple rounds over days. The path for educational material, chiefly distilled YouTube-transcript learnings. Full completion tethers; failure reschedules and extends.

The distinction is deliberate and philosophical: an asserted fact enters the corpus because the human *judged it true*; learned material enters because the human *demonstrably absorbed it*. Material you never engaged with never pollutes the knowledge base. This extends ADR 0001 with a second gate rather than weakening the first — both paths still end in a human-earned trust decision, just earned differently.

It is hard to reverse (the second gate is baked into the Memory lifecycle and the retrieval-eligibility rule) and surprising (a memory that becomes trusted only after you pass spaced quizzes is unusual), which is why it is recorded. What tethers is the **distilled learnings** of a source, not the raw transcript; the raw transcript stays in the read-only ingested corpus as provenance.
